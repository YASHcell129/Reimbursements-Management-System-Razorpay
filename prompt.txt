I am building a backend-only Reimbursements Management System as part of a Razorpay hiring assignment.

Tech Stack:

* Node.js
* Express.js
* PostgreSQL
* JavaScript (No TypeScript)
* JWT Authentication
* Cookie-based Authentication
* bcrypt for password hashing

Project Requirements:

Roles:

1. EMP (Employee) - default role for new registrations
2. RM (Reporting Manager)
3. APE (Accounts Payable Executive)
4. CFO (Root User)

Important Rules:

* Every new user registers as EMP.
* Only emails ending with @org.com are allowed.
* CFO account is seeded and cannot register.
* Every employee reports to exactly one RM.
* A reimbursement becomes APPROVED only when:

  * RM approves it
  * AND one APE approves it

Required APIs:

Public APIs:
POST /rest/onboardings/register
POST /rest/onboardings/login
POST /rest/onboardings/logout

Protected APIs:
POST /rest/roles/assign
GET /rest/employees
POST /rest/employees/assign
DELETE /rest/employees/assign
POST /rest/reimbursements
PATCH /rest/reimbursements
GET /rest/reimbursements
GET /rest/reimbursements/:userId

Authentication:

* Login should generate JWT.
* JWT should be stored in an HTTP-only cookie named "auth".
* Protected APIs must read the cookie automatically.
* Implement authentication middleware.
* Implement role-based authorization middleware.
